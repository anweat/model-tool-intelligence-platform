# 模型工具调用智能分析平台项目说明

## 项目定位

本项目面向多模型应用开发场景，统一管理模型供应商、API Key、模型目录与调用日志，并基于工具调用数据自动生成系统画像、工具价值分析、成本风险、缓存收益和上线可行性评估。

项目不是单纯的 API Key 管理工具，核心创新点是把模型调用日志转化为可决策的信息，帮助团队回答以下问题:

- 当前系统主要依赖哪些工具和模型能力。
- 哪些工具调用频率高、成功率高、缓存收益明显。
- 现有调用成本是否接近预算。
- 当前模型系统是否适合上线或试点。
- 如果扩大到指定月调用量，费用是否可控。

## 核心功能

1. 供应商管理
   - 维护模型供应商名称、Base URL、状态和备注。
   - 支持启用/停用供应商。

2. API Key 管理
   - 记录 API Key 标签、负责人、所属供应商和状态。
   - 前端只展示脱敏后的 Key。

3. 模型目录管理
   - 维护模型上下文窗口、输入单价、输出单价、缓存折扣和工具调用能力。
   - 支持停用不再参与调用统计的模型。

4. 工具调用日志
   - 登记工具名称、模型、Token 输入输出、缓存命中、耗时、成功状态和失败类型。
   - 支持删除或归档调用记录。

5. 智能统计与可行性评估
   - 生成缓存命中率、成功率、平均延迟、费用和预算占用。
   - 自动识别主场景、主供应商、主模型和工具分布。
   - 对工具计算调用次数、成功率、缓存命中率、平均耗时、总费用和价值等级。
   - 按成功率、缓存命中率、延迟、预算占用和供应商集中度计算可行性评分。
   - 生成上线建议和治理建议。

6. 方案测算
   - 输入预计月调用次数、平均输入输出 Token、目标缓存命中率和模型。
   - 输出预计月费用、预算占用和可行性建议。

7. 审计与导出
   - 记录新增、修改、删除、归档等操作。
   - 支持导出完整 JSON 报表，包含状态、统计摘要、智能洞察和审计日志。

## 创新点

传统管理系统通常只记录调用次数和费用。本项目进一步把工具调用数据作为系统运行证据，生成面向决策的分析结论:

- 系统画像: 从调用日志推导系统主要业务场景、工具数量、模型数量和主模型。
- 工具价值: 结合调用频率、成功率、缓存率、耗时和费用判断工具价值等级。
- 可行性评分: 用量化规则评估系统是否适合上线、试点或需要优化。
- 成本预测: 使用模型单价、缓存折扣和预计调用量估算扩容后的月费用。
- 治理建议: 自动提示缓存、重试、限流、模型切换和供应商备份等优化方向。

## REST API

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/health` | 服务健康检查 |
| GET | `/api/state` | 获取前端所需状态、摘要和智能洞察 |
| GET | `/api/insights` | 获取系统画像、可行性评分和工具洞察 |
| POST | `/api/feasibility` | 根据模型和预计调用量进行费用测算 |
| GET | `/api/export` | 导出完整报表 |
| POST | `/api/providers` | 新增供应商 |
| POST | `/api/keys` | 新增 API Key |
| POST | `/api/models` | 新增模型 |
| POST | `/api/calls` | 登记调用记录 |
| PATCH | `/api/{providers|keys|models|calls}/{id}` | 修改资源 |
| DELETE | `/api/{providers|keys|models|calls}/{id}` | 删除资源 |
| PATCH | `/api/settings` | 修改预算、告警阈值和保留天数 |
| POST | `/api/calls/archive-expired` | 按保留天数归档过期调用 |
| POST | `/api/reset` | 重置演示数据 |

## 验收标准

- 可以新增和管理供应商、API Key、模型和调用记录。
- API Key 在列表中必须脱敏展示。
- 登记调用后，Token、费用、缓存命中率和工具摘要实时更新。
- 系统能够生成可行性评分、系统画像和治理建议。
- 可根据预计月调用量进行费用测算。
- 可以导出包含统计与洞察的 JSON 报表。
- `npm test` 冒烟测试通过。
