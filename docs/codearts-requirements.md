# CodeArts 需求录入清单

## Epic

Epic: 统一模型供应商 API Key 与调用成本管理平台

## Feature 与 User Story

### Feature 1: 供应商管理

US-01: 作为系统管理员，我希望登记模型供应商的名称、接口地址和状态，以便统一维护可用供应商。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 可以新增供应商并在列表中展示名称、Base URL 和状态。
  - 新增供应商后可以被 API Key、模型和调用记录选择。

US-02: 作为系统管理员，我希望查看每个供应商下的模型数量，以便判断供应商接入是否完整。
- 优先级: 中
- 重要程度: 中
- 验收标准:
  - 供应商列表显示关联模型数量。
  - 模型新增后数量自动更新。

### Feature 2: API Key 管理

US-03: 作为系统管理员，我希望按供应商保存 API Key 标签、负责人和密钥，以便集中管理调用凭证。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 可以提交供应商、标签、负责人和密钥。
  - Key 保存后持久化到本地数据文件。

US-04: 作为安全负责人，我希望列表中只展示脱敏后的 Key，以降低敏感信息泄露风险。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - Key 列表不展示完整密钥。
  - 前后端接口返回结果包含 maskedSecret 而不是完整 secret。

US-05: 作为系统管理员，我希望记录 API Key 的启用状态，以便区分可用和停用凭证。
- 优先级: 中
- 重要程度: 中
- 验收标准:
  - Key 数据包含 enabled/disabled 状态。
  - Key 列表展示状态标签。

### Feature 3: 模型目录与价格管理

US-06: 作为系统管理员，我希望维护模型名称、上下文窗口和工具调用能力，以便选择合适模型。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 可以为指定供应商新增模型。
  - 模型列表展示上下文窗口和是否支持工具调用。

US-07: 作为财务统计人员，我希望维护模型输入与输出单价，以便估算每次调用成本。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 模型表单支持输入输入单价和输出单价。
  - 调用记录能根据模型价格计算费用。

US-08: 作为性能优化人员，我希望维护缓存命中折扣，以便统计缓存带来的费用变化。
- 优先级: 中
- 重要程度: 中
- 验收标准:
  - 模型支持设置 0 到 1 的缓存折扣。
  - 缓存命中调用按折扣后的输入 Token 计费。

### Feature 4: 调用记录与缓存统计

US-09: 作为开发人员，我希望登记一次模型调用的模型、Token、耗时和工具名称，以便后续分析使用情况。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 调用登记表单包含供应商、模型、工具、输入 Token、输出 Token、耗时。
  - 提交后调用记录进入最近调用列表。

US-10: 作为开发人员，我希望标记调用是否缓存命中，以便计算缓存命中率。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 调用登记表单包含缓存命中选项。
  - 总览区实时显示缓存命中率。

### Feature 5: 统计报表与计费汇总

US-11: 作为项目负责人，我希望查看总调用次数、输入输出 Token 和估算费用，以便掌握项目运行成本。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 总览区展示调用次数、输入 Token、输出 Token、估算费用。
  - 新增调用后统计数据自动刷新。

US-12: 作为项目负责人，我希望查看工具调用摘要，以便了解模型主要服务于哪些业务工具。
- 优先级: 中
- 重要程度: 中
- 验收标准:
  - 后端 summary 返回 toolUsage 聚合结果。
  - 最近调用列表展示每条调用对应的工具名称。

US-13: 作为财务统计人员，我希望设置月度预算和告警阈值，以便在费用接近预算时及时处理。
- 优先级: 中
- 重要程度: 高
- 验收标准:
  - 系统支持配置月度预算、告警阈值和日志保留天数。
  - 当估算费用达到阈值时，总览区显示预算告警状态。

US-14: 作为项目负责人，我希望导出调用与计费报表，以便提交审计或课程报告材料。
- 优先级: 中
- 重要程度: 中
- 验收标准:
  - 系统提供导出 JSON 报表功能。
  - 导出内容包含供应商、模型、调用记录、统计汇总和导出时间。

### Feature 6: 智能洞察与可行性评估

US-15: 作为项目负责人，我希望系统根据工具调用日志生成主业务场景、主供应商和主模型，以便快速理解系统运行特征。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 系统画像展示工具数量、供应商数量、模型数量、主业务场景、主供应商和主模型。
  - 新增调用记录后系统画像自动更新。

US-16: 作为研发负责人，我希望按工具统计成功率、缓存命中率、耗时和费用，以便判断哪些工具值得继续投入。
- 优先级: 高
- 重要程度: 高
- 验收标准:
  - 工具洞察列表展示每类工具的调用次数、成功率、缓存命中率、平均耗时和费用。
  - 系统能给出高价值、观察、需治理等工具价值等级。

US-17: 作为财务统计人员，我希望输入预计调用量和 Token 规模后获得月度费用测算，以便判断方案是否可控。
- 优先级: 中
- 重要程度: 高
- 验收标准:
  - 用户可以选择模型并输入预计月调用次数、平均输入 Token、平均输出 Token 和目标缓存命中率。
  - 系统返回预计费用、预算占用比例和可行性建议。

US-18: 作为系统运维人员，我希望系统根据失败率、缓存率、延迟和预算占用生成治理建议，以便优化上线方案。
- 优先级: 中
- 重要程度: 高
- 验收标准:
  - 系统根据成功率、缓存命中率、平均延迟、预算占用和供应商集中度计算可行性评分。
  - 系统输出上线、试点、优化或暂缓建议，并给出具体治理方向。

## 非功能需求

- 安全性: API Key 不应在前端完整展示；导出报表不包含完整 secret。
- 可审计性: 新增、修改、删除、归档等操作写入审计日志。
- 可维护性: 系统使用清晰的 REST API，核心统计逻辑集中在后端。
- 可演示性: 项目无需外部数据库，启动后可使用内置演示数据完成验收。
- 可扩展性: 后续可替换 JSON 文件为 SQLite、PostgreSQL 或 MySQL，并引入登录权限。

## 迭代计划

- Sprint 1: 供应商、API Key、模型目录和调用登记。
- Sprint 2: Token、费用、缓存命中率、工具摘要和预算告警。
- Sprint 3: 系统画像、工具价值、可行性评分、方案测算、审计日志和报表导出。

## 风险与应对

- 密钥泄露风险: 前端脱敏展示，后续版本应增加加密存储。
- 费用估算偏差: 当前使用手工维护单价，后续可接入供应商价格 API 或定期校准。
- 样本量不足: 小样本下评分仅作为辅助判断，后续需要按时间窗口和业务场景分组评估。
- 单供应商依赖: 可行性评分纳入供应商集中度，提示配置备用模型。

## 利益相关者

- 系统管理员: 维护供应商、模型、API Key 和基础配置。
- 开发人员: 登记模型调用、查看 Token 和缓存命中情况。
- 安全负责人: 关注密钥脱敏、负责人归属和凭证状态。
- 财务/项目负责人: 查看调用成本、缓存收益和模型使用趋势。
- 教师/助教: 通过 CodeArts 项目和报告验收需求、建模与 MVP 实现情况。

## 用户画像

画像 1: 学生开发者
- 目标: 快速接入多个模型供应商，并清楚知道每次调用的成本。
- 痛点: Key 分散、模型价格不统一、Token 统计依赖手工整理。

画像 2: 项目负责人
- 目标: 以低成本完成模型能力验证，并保留可审计的调用数据。
- 痛点: 不同模型费用难比较，缓存效果缺少量化指标。

画像 3: 安全负责人
- 目标: 降低 API Key 泄露风险，保证密钥使用可追踪。
- 痛点: 明文密钥散落在代码或个人文档中，缺少负责人信息。
