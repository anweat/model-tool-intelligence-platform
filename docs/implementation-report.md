# 编码实现报告

## 项目简介

本项目名为“模型工具调用智能分析平台”，用于统一管理不同模型供应商提供的 API Key、模型列表和工具调用日志，并将调用数据转化为系统画像、工具价值分析、缓存收益、成本风险和上线可行性评估。系统已实现供应商管理、API Key 脱敏展示、模型价格维护、调用登记、缓存命中率计算、输入输出 Token 统计、工具调用摘要、估算计费、预算告警、方案测算、审计日志和报表导出。

## 使用的 Prompt

我们需要做一个用于完成 CodeArts 作业的项目，主题是统一管理各模型提供商提供的 API Key 和模型列表，做简要的缓存命中计算、输入输出统计、模型调用工具总结和计费。请先尝试登录和进行 CodeArts 操作，再实现一个具备数据持久化的 MVP，并形成可录入 CodeArts 的需求和建模材料。

## Vibe Coding 过程

1. 读取作业 PDF，提取需求管理、软件建模、MVP、报告提交要求。
2. 尝试打开华为云 CodeArts 控制台和产品页。内置浏览器访问项目页存在超时问题，随后改用本地 CodeArts Agent 登录态和华为云官方 Python SDK 操作 CodeArts Req。
3. 根据作业要求将项目拆分为 1 个 Epic、6 个 Feature 和 18 个 User Story。
4. 实现 Node.js 应用，采用内置 HTTP 服务和 `data/store.json` 文件持久化，避免外部数据库或包下载依赖。
5. 将创新点从普通账本扩展为“基于工具调用数据的系统可行性评估”，补充系统画像、工具价值等级、失败类型分析、预算风险和扩容测算。
6. 补充用例图、活动图、类图、顺序图的 Mermaid 建模文本，便于导出图片或在 CodeArts 软件建模中复刻。

## AI 生成后人工调整

- 将项目范围控制为课程 MVP，优先保证关键业务流程可运行。
- API Key 在接口返回时脱敏，避免列表展示完整密钥。
- 计费逻辑区分输入 Token、输出 Token 与缓存命中折扣。
- 增加预算阈值、缓存节省、按供应商/模型/工具聚合统计。
- 增加启停、删除和导出接口，使系统更接近完整管理项目。
- 增加 `/api/insights` 和 `/api/feasibility`，用于生成系统画像、工具洞察和方案费用测算。
- 增加审计日志、归档过期调用、失败类型统计和 `npm test` 冒烟测试。
- 文档中的 Feature 和 User Story 与代码功能逐项对应。

## 已知问题

- 当前 MVP 使用 JSON 文件作为本地持久化，适合作业演示；生产环境应替换为 SQLite、PostgreSQL 或 MySQL，并对密钥加密存储。
- 目前未实现完整登录认证和多角色权限隔离；课程 MVP 通过本地运行演示。
- CodeArts 工作项已通过 SDK 自动创建；网页端截图仍建议用户在浏览器中打开项目后截取。
